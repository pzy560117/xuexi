需要先了解[【外部】Prompt2Repo项目准入手册](https://j0t9xglvod.feishu.cn/wiki/Upjzwe0P9ipTWWkrRk9cZApYnGe)里的具体验收标准，再根据具体项目来判断这个项目是否满足执行手册里的验收标准。

>
> 格式要求：统一用md格式文件，需要纯文本
>
> 准入考核次数没有限制，可重复提交，通过即可，目的是为了让大家了解项目规则以及流程
>
> 考核结果：在项目准入查看最终结果

# 提交文档要求

 **格式要求：** **统一用md格式文件 需要纯文本**

基于项目准入手册里的验收标准，对项目进行检查。提交的答案需要包含下面每个维度的说明

每一项都需要检查，并在报告里指出

**针对所有题目的通用要求**

1. 硬性门槛
   1. 交付产物是否能够实际运行和验证
   2. 交付产物是否严重偏离 Prompt 主题
2. 交付完整性
   1. 交付产物是否完整覆盖 Prompt 中明确提出的核心需求
   2. 交付产物是否具备从 0 到 1 的基本交付形态，而非仅提供局部功能、示意性实现或片段代码。
3. 程与架构质量
   1. 交付产物在当前问题规模下是否采用了合理的工程结构与模块划分
   2. 交付产物是否体现出基本的可维护性与可扩展性意识，而非临时性或堆叠式实现。
4. 工程细节与专业度
   1. 交付产物在工程细节与整体形态上是否体现出专业工程实践水准，包括但不限于错误处理、日志、校验、接口设计
   2. 交付产物是否具备真实产品或服务应有的功能组织形态，而非停留在示例或演示级实现。
5. Prompt 需求理解与适配度
6. 交付产物是否准确理解并响应了 Prompt
7. 美观度（仅限全栈、纯前端题目）
   1. 交付产物视觉 / 交互是否互贴合场景，且设计美观
