### 如果 prompt 是 英文的，所有产物附件都应该以英文形式呈现，包括轨迹对话，注释，日志，和各种相关文档，UI显示，等等内容全部需要使用英文，禁止出现中文字项(自测报告不需要打包到产物不要求中文)

### 辅助目录结构检查脚本 **validate_package.py**

执行方式 python validate_package.py TASK-**路径

添加--repair会自动删除缓存/多余文件，按照规则进行重命名，按照规定文件位置部署目录，执行删除操作的文件会被备份到输入路径根目录下的.backup文件夹下

添加 --convert-legacy 会执行从旧目录结构转变为新目录结构操作，会有执行变动规划日志输出在终端需要二次确认

运行--repair前，会有执行删除日志输出在终端，二次判定输入yes确认。确保检查过删除日志，防止删除项目需要文件

### Claude code 轨迹转换脚本**merge_claude_subagents_trajectory.py**

具体用法见脚本头部说明

1. 产物附件打包不可以 环境依赖的相关内容，比如 node_modules/.venv/.net等等

4. 说明性文档readme.md，介绍项目实现功能，启动，测试，架构等内容
5. 压缩包命名 为 TASK-**，使用题目ID 作为交付物命名即可，项目目录命名全部为repo，比如fullstack/pure_backend/mobile_app等等这些信息都转移到metadata.json中去，与工程项目产物同级应该包括产出相关的docs文档，prompt.md(原始的题目),sessions文件夹，不管是单一的还是多个trajectory.json文件都放在这个sessions文件夹，questions.md(记录自己在生产过程中认为AI出现的问题)
