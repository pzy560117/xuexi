# MediaCrawler 小白原子化教程

> **目标读者**：完全没有编程经验的用户  
> **预计耗时**：30 分钟可跑通第一次爬取  
> **原则**：每个步骤只做一件事，每步都有「✅ 验证方法」

---

## Step 1：安装 Python 环境管理工具 `uv`

> `uv` 是目前最快的 Python 包管理工具，项目推荐使用

**Windows 用户**：打开 PowerShell，执行：
```powershell
irm https://astral.sh/uv/install.ps1 | iex
```

**macOS / Linux 用户**：
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**✅ 验证**：关闭并重新打开终端，输入：
```powershell
uv --version
```
看到版本号（如 `uv 0.7.x`）即为成功。

---

## Step 2：安装 Node.js

> 抖音和知乎的签名需要 Node.js（其他平台也建议装上）

1. 打开 [https://nodejs.org](https://nodejs.org/en/download/)
2. 下载 **LTS 版本**（Windows 选 Installer .msi）
3. 双击安装，一路 Next 即可

**✅ 验证**：
```powershell
node --version
```
看到 `v16.x` 以上版本即可。

---

## Step 3：下载项目代码

**方式一（推荐）**：使用 Git
```powershell
git clone https://github.com/NanmiCoder/MediaCrawler.git
cd MediaCrawler
```

**方式二**：手动下载
1. 打开 [项目 GitHub 页面](https://github.com/NanmiCoder/MediaCrawler)
2. 点击绿色 `Code` → `Download ZIP`
3. 解压到你想要的目录
4. 在终端中 `cd` 到该目录

**✅ 验证**：
```powershell
dir main.py   # Windows
ls main.py    # macOS / Linux
```
看到 `main.py` 文件即表示目录正确。

---

## Step 4：安装 Python 依赖

在项目目录下执行：
```powershell
uv sync
```

> ⏳ 首次运行会自动下载 Python 3.11+ 和所有依赖包，约需 2-5 分钟

**✅ 验证**：
```powershell
uv run python --version
```
看到 `Python 3.11.x` 或更高版本即可。

---

## Step 5：安装浏览器驱动

```powershell
uv run playwright install
```

> ⏳ 会下载 Chromium 浏览器驱动，约 150MB

**✅ 验证**：命令执行完毕无报错即可。

---

## Step 6：（可选）修改配置

打开 `config/base_config.py`，按需修改：

```python
# 1. 选择目标平台（默认小红书）
PLATFORM = "xhs"  # xhs | dy | ks | bili | wb | tieba | zhihu

# 2. 设置搜索关键词（英文逗号分隔）
KEYWORDS = "编程副业,编程兼职"

# 3. 设置爬取数量
CRAWLER_MAX_NOTES_COUNT = 15

# 4. 是否爬取评论（默认开启）
ENABLE_GET_COMMENTS = True

# 5. CDP 模式（推荐开启，反检测能力更强）
ENABLE_CDP_MODE = True
```

> 💡 如果不想改配置文件，也可以通过命令行参数覆盖，见 Step 7。

## Step 7：🚀 首次运行

### 场景一：搜索小红书笔记（最常用）

```powershell
uv run main.py --platform xhs --lt qrcode --type search --keywords "美食推荐"
```

运行后会发生什么：
1. 🖥️ 弹出一个浏览器窗口
2. 📱 屏幕上显示小红书登录二维码
3. 👉 **打开手机小红书 APP → 扫描二维码 → 确认登录**
4. 🔄 浏览器自动开始搜索和爬取
5. 📁 数据保存到 `data/xhs/` 目录下

### 场景二：爬取抖音内容

```powershell
uv run main.py --platform dy --lt qrcode --type search --keywords "Python教程"
```

### 场景三：爬取 B 站视频信息

```powershell
uv run main.py --platform bili --lt qrcode --type search --keywords "编程教学"
```

**✅ 验证**：查看 `data/` 目录下是否生成了 `.jsonl` 文件。

## Step 8：查看爬取结果

### JSONL 格式（默认）

```powershell
dir data\xhs\    # Windows
ls data/xhs/     # macOS / Linux
```

每行一个 JSON 对象，包含：
- **内容文件**：标题、正文、点赞数、评论数、分享数 等
- **评论文件**：评论内容、评论者昵称、点赞数 等
- **创作者文件**：昵称、粉丝数、作品数 等

### 切换为 Excel 格式（更直观）

```powershell
uv run main.py --platform xhs --lt qrcode --type search --save_data_option excel
```

> 数据会保存为 `.xlsx` 文件，双击即可用 Excel/WPS 打开。

## Step 9：进阶用法

### 📌 按指定帖子 ID 爬取

```powershell
uv run main.py --platform xhs --lt qrcode --type detail --specified_id "帖子URL或ID"
```

### 👤 爬取指定创作者

```powershell
uv run main.py --platform xhs --lt qrcode --type creator --creator_id "创作者URL"
```

### 💬 爬取二级评论

```powershell
uv run main.py --platform xhs --lt qrcode --type search --get_comment true --get_sub_comment true
```

### 🖥️ 使用 WebUI 可视化界面

```powershell
uv run uvicorn api.main:app --port 8080 --reload
```

然后打开浏览器访问 `http://localhost:8080`，可视化操作全部功能。

### 📊 使用数据库存储

```powershell
# 1. 先初始化数据库
uv run main.py --init_db sqlite

# 2. 使用 SQLite 存储
uv run main.py --platform xhs --lt qrcode --type search --save_data_option sqlite
```

### ☁️ 生成词云图

在 `config/base_config.py` 中设置：
```python
ENABLE_GET_WORDCLOUD = True
ENABLE_GET_COMMENTS = True
```
然后使用 json 保存模式运行：
```powershell
uv run main.py --platform xhs --lt qrcode --type search --save_data_option json
```

### 🔢 控制爬取参数

```powershell
# 指定自定义关键词 + 限制数量 + 并发数
uv run main.py --platform xhs --lt qrcode --type search \
    --keywords "旅游攻略,拍照技巧" \
    --max_concurrency_num 2
```

## Step 10：常见问题排查

### ❌ 问题 1：`SyntaxError: 缺少 ';'`
**原因**：没装 Node.js  
**解决**：回到 Step 2 安装 Node.js

### ❌ 问题 2：小红书扫码后滑块验证一直失败
**解决**：
1. 确保 `HEADLESS = False`（显示浏览器窗口）
2. 删除项目目录下的 `brower_data/` 文件夹，重新登录
3. 手动在弹出的浏览器中通过滑块验证

### ❌ 问题 3：`TimeoutError: Timeout 30000ms exceeded`
**原因**：网络问题（可能开了 VPN/代理）  
**解决**：关闭 VPN 或代理后重试

### ❌ 问题 4：爬了一段时间后数据拿不到了
**原因**：账号触发了平台风控  
**解决**：
1. 降低爬取数量：在 `config/base_config.py` 中设置 `CRAWLER_MAX_NOTES_COUNT = 5`
2. 增加间隔：设置 `CRAWLER_MAX_SLEEP_SEC = 5`
3. 等待一段时间后再试

### ❌ 问题 5：想换账号登录
**解决**：删除项目目录下的 `brower_data/` 文件夹

### ❌ 问题 6：浏览器检测失败（CDP 模式）
**解决**：
1. 确保已安装 Chrome 或 Edge 浏览器
2. 如果浏览器路径不标准，在 `config/base_config.py` 中手动设置：
   ```python
   CUSTOM_BROWSER_PATH = r"C:\Program Files\Google\Chrome\Application\chrome.exe"
   ```
3. 或者关闭 CDP 模式使用标准模式：`ENABLE_CDP_MODE = False`

## 📂 数据存储方式一览

| 存储方式 | 命令参数 | 输出位置 | 特点 |
|---------|---------|---------|------|
| JSONL（默认） | `--save_data_option jsonl` | `data/{platform}/` | 性能最好，追加写入 |
| JSON | `--save_data_option json` | `data/{platform}/` | 结构完整 |
| CSV | `--save_data_option csv` | `data/{platform}/` | 简单通用 |
| Excel | `--save_data_option excel` | `data/{platform}/` | 可视化，适合分析 |
| SQLite | `--save_data_option sqlite` | `database/sqlite_tables.db` | 轻量数据库 |
| MySQL | `--save_data_option db` | MySQL 服务器 | 生产级数据库 |
| PostgreSQL | `--save_data_option postgres` | PostgreSQL 服务器 | 高级数据库 |
| MongoDB | `--save_data_option mongodb` | MongoDB 服务器 | NoSQL 数据库 |
