# CLAUDE.md — 全局通用配置

## 通用原则

- 代码质量：函数 < 50 行，文件 < 800 行
- 结构化日志，禁止裸 console.log / print
- 输入校验 + 错误处理 + 统一错误响应
- 不暴露 token/密码/密钥到日志或前端

## Superpowers 插件已安装

本项目已启用超级自动化流水线。**请彻底放弃手动分阶段调用（如单独调用 brainstorming）。**

## 全自动流水线触发规则（最高优先级）

当用户在会话中输入任何“业务需求、功能需求、或题目描述”时，**必须直接进入全自动流程，绝对不允许在某个环节（环境）完成后停下不继续**：

1. **原封不动**将用户给到的完整需求提示词写入当前目录的 `prompt.md` 中（UTF-8编码）。
2. 自动生成 `TASK-ID`（格式：`TASK-YYYYMMDD-HHMMSS`）。
3. **立即一键执行全自动子代理流水线（具备多阶段自研能力）**：
   ```bash
   /superpowers:idea2repo "请严格基于 prompt.md 中的业务需求进行架构设计与全流程开发" --task-id <TASK-ID>
   ```

**执行约束：**

- **严禁**擅自缩水流程。必须通过 idea2repo 流水线，依次调用内置的子代理（Architecture Research, BDD Review 等）一次性跑完解析、计划、执行、自测和打包全过程！
- **禁止**在流程启动前或中间弹出“A/B/C 选择菜单”阻塞用户，由代理子代理自动分析并选择最佳推荐项继续！
- 必须遵循 `AGENTS.md` / `CLAUDE.md` 的规范优先级。
- 不得使用 mock 数据替代真实逻辑。
- 关键结论需给出可追溯证据。
